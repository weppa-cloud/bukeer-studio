# Theme References — Hardcoded Design Templates

This directory contains **hardcoded theme references** — standalone Next.js sites
generated by external AI tools (V0.dev, Bolt.new, Claude) that serve as the
**design source of truth** for Bukeer themes.

## Purpose

These files are NOT production code. They are design references used during the
theme creation workflow:

```
1. DESIGN (external)     → Generate hardcoded theme with AI tool
2. REFERENCE (this dir)  → Store it here as permanent design reference
3. MAP (Claude Code)     → Create section variants in Bukeer components
4. BRIDGE (auto)         → CSS Variable Bridge maps tokens automatically
5. VALIDATE (MCP)        → Compare Bukeer output vs hardcoded reference
6. SAVE (RPC)            → save_website_as_template_v2()
```

## Structure

```
themes/references/
  {theme-name}-{mood}/
    app/                    # Pages (home, listings, details)
    components/             # Standalone components (cards, sections)
    lib/                    # Utilities, motion presets
    app/globals.css         # Design tokens (CSS variables, colors, fonts)
```

## Key Files Per Theme

| File | Purpose |
|------|---------|
| `app/globals.css` | CSS variables, color palette, fonts, animations |
| `app/page.tsx` | Homepage section composition order |
| `components/HotelCard.tsx` | Card design reference |
| `app/hoteles/page.tsx` | Listing page layout + filters |
| `app/hoteles/[slug]/page.tsx` | Detail page layout |

## CSS Variable Bridge

The M3ThemeProvider in `lib/theme/m3-theme-provider.tsx` automatically generates
bridge CSS variables that map shadcn tokens to the variable names used in these
reference themes:

| Reference Variable | Bridge Source |
|-------------------|--------------|
| `var(--bg)` | `hsl(var(--background))` |
| `var(--accent)` | `hsl(var(--primary))` |
| `var(--bg-card)` | `hsl(var(--card))` |
| `var(--text-heading)` | `hsl(var(--foreground))` |
| `var(--border-subtle)` | `hsl(var(--border) / 0.5)` |
| `var(--card-badge-bg)` | `hsl(var(--background) / 0.7)` |

This means section components can use `style={{ color: "var(--accent)" }}`
and it works with ANY Bukeer theme — not just the one it was designed for.

## Creating a New Theme

1. Generate theme in V0.dev/Bolt.new with the stack: Next.js + Tailwind + Framer Motion
2. Copy source files to `themes/references/{name}-{mood}/`
3. Use Claude Code to map sections to Bukeer variants
4. Validate with Chrome DevTools MCP comparing side by side
5. Save as template via `save_website_as_template_v2()`
