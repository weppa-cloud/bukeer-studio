# Sprint 0.25B: Bukeer Design Lock Code-First & Human-Agent Prototype

> Approved: 2026-05-18
> Revised: 2026-05-18
> Duration: 4 weeks
> Calendar: 2026-05-18 to 2026-06-12
> Parent epic: `weppa-cloud/bukeer-flutter#823`
> Sprint type: code-first design lock + prototype readiness, not production build.
> Design source of truth: v0 registry + Bukeer tokens + Next component contract + prototype. Figma is optional support, not a blocker.

## Objective

Convert the approved Human-Agent GUI direction into a validated, implementable, code-first foundation for Bukeer Admin Next.

This sprint replaces the previous Figma-heavy path with a Design System Lock built from:

- v0 registry-backed concepts.
- Bukeer signature tokens.
- React/Tailwind/shadcn component contracts.
- A prototype-ready Next implementation backlog.

Approved direction:

- Planner Workbench V2 is the primary direction.
- Conversation Copilot V2 is the CRM/messaging support pattern.
- Trace Approval V2 is the governance support pattern.
- Itinerary Builder V2 is not approved as-is; it must be rebuilt as Itinerary Manifest V3.

## Sprint Outcomes

By 2026-06-12, the team should have:

1. Approved Agentic UI State Model.
2. Registry-backed v0 V3 outputs.
3. Bukeer Signature Direction selected: one primary direction and one alternative.
4. Design System Lock Code-First: tokens, component contracts, state variants and responsive rules.
5. Beta partner feedback on Planner Workbench.
6. Next component contract ready for implementation.
7. Implementation backlog for the first Next prototype sprint.

## Scope

In scope:

- Agentic UI state semantics.
- Registry-backed v0 V3 generation.
- Bukeer Design System Lock Code-First.
- Tokens for Next/Tailwind: color, typography, spacing, radius, borders, elevation, density and agentic state roles.
- Component contract for AdminShell, PlannerWorkbench, ConversationCopilot, TraceApprovalPanel, ItineraryManifest and ManagerControlPlane.
- Responsive rules for desktop, tablet and mobile.
- Planner Workbench beta review.
- Next component contract refinement.
- Implementation backlog for a non-production prototype.
- Optional Figma snapshots only when needed for external review or beta partner comments.

Out of scope:

- Production AdminShell implementation.
- Real writes.
- Public customer send.
- Payment/reservation actions.
- Supplier mutations.
- Replacing Flutter routes.
