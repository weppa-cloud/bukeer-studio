# Prompt 04: Conversation Copilot

Copy/paste this full prompt into v0.

---

Create a desktop-first React + TypeScript + Tailwind + shadcn/Radix UI concept for Bukeer "Conversation Copilot".

Context:

- Internal travel agency CRM/conversation workspace.
- Human agents respond to WhatsApp/web/email leads.
- AI can summarize, classify intent, suggest replies, identify missing data, and propose next steps.
- AI must not send public messages by default. Human must review and send.

Layout:

- Left conversation inbox with filters: unread, assigned to me, high intent, waiting on client, AI blocked.
- Center conversation thread with message history, private notes, customer profile and reply composer.
- Right copilot panel with summary, intent, missing data, suggested reply, itinerary opportunity and trace.
- Topbar within screen: assignment, status, SLA, source, permission badges.

Sample conversation:

- Customer: Mariana Rios.
- Channel: WhatsApp.
- Request: "Somos 2 adultos y 2 ninos, queremos Cartagena en julio, hotel con playa y actividades suaves."
- AI identifies: family trip, high intent, missing children ages, budget confirmed, flexible payment interest.

Required features:

- Suggested reply card with editable message.
- Button actions: insert draft, edit, send after human review, reject suggestion.
- Qualification panel: trip type, party size, destination, dates, budget, urgency.
- Missing-data chips.
- Create itinerary draft action, currently blocked until missing data resolved.
- Private note generated by AI, not public.
- Trace timeline: input messages, context packet, tool call, output, human decision.
- Permission lock for "reply_any" or "force_reopen" action.

Required states:

- Normal: active conversation and suggested reply.
- Loading: AI composing reply and checking catalog context.
- Empty: no conversations in filter.
- Error: Chatwoot/Supabase sync failed.
- No permission: user cannot reply to conversation assigned to another agent.
- AI suggestion: reply draft and qualification.
- AI blocked: blocked public send because human review is required.
- Approval required: force reopen or macro execution requires permission/approval.

Design requirements:

- Operational, dense, quick to scan.
- No landing/hero/marketing style.
- Use shadcn/Radix primitives and lucide-react icons.
- Composer must make clear what is public vs private.
- AI suggestion must show rationale and confidence.
- AI blocked must be visually distinct from technical error.
- Responsive: inbox, thread and copilot become tabs or stacked panels.
- Include accessible labels and focus states.

Output:

- One complete Conversation Copilot screen with mock data.
- Include component structure that can later map to Bukeer Admin Next.
- No backend calls.
