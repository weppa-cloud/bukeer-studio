# Sprint 0.25D: Read-Only Agentic Admin Foundation

> Date: 2026-05-18
> Parent epic: `weppa-cloud/bukeer-flutter#823`
> Input: approved Bukeer Signature Planner Workbench + Trace Inspector
> Goal: move from fixture-first prototype to real read-only foundation without production writes.

## Objective

Connect the approved Signature UI direction to a safe, auditable admin foundation:

- read-only data contracts;
- Domain Tool Registry in readonly mode;
- AgentRun / ToolInvocation / Approval ledger schemas;
- server-side read adapters;
- protected Planner Workbench route;
- no Supabase writes, no public sends, no supplier holds, no payments.

This sprint is not the Flutter replacement. It is the bridge from visual approval to implementation readiness.

## Week 1: Contracts And Registry

Deliverables:

- Expand `@bukeer/admin-contract` with AgentRun, ToolInvocation and Approval ledger schemas.
- Add read-only Domain Tool Registry metadata for planner, opportunity, itinerary and trace capabilities.
- Keep registry independent from Supabase execution.
- Add tests proving prototype tools are readonly.

Acceptance:

- Contracts parse valid fixture data and reject unsafe writes.
- Tool metadata can be listed and looked up by ID.
- Registry has no `write`, `mutation`, `payment`, `public_send` or supplier hold capability.

## Week 2: Read-Only Data Adapter

Deliverables:

- Server-side read adapter for planner opportunities.
- Normalize real data into Travel Ontology v1 shapes.
- Preserve fixture fallback for local/dev review.
- Add permission gate through `getAdminSessionContext()`.

Acceptance:

- Route can run fixture-first and real-read-only behind a flag.
- No service role, write-capable client or mutation reaches client code.
- Empty/error/no permission states render through Signature UI.

## Week 3: Trace And Approval Ledger UI

Deliverables:

- Map AgentRun / ToolInvocation / Approval rows into Trace Inspector.
- Show permission, policy, data used, confidence and timeline.
- Add approval simulation states: approved, rejected, executing, executed.

Acceptance:

- Every AI suggestion and blocked action opens a Trace Inspector.
- Human approval boundary remains visible in light/dark.
- No action button can imply a production write.

## Week 4: Hardening And Beta Readiness

Deliverables:

- Responsive QA at 1440, 1280, 834 and 390px.
- Playwright smoke for light/dark and inspector.
- Build and typecheck gates.
- Beta review packet using approved screenshots and scenarios.
- Epic #823 progress comment with readiness percentage and evidence.

Acceptance:

- Planner can understand trip context, missing data, AI suggestion, margin risk and approval state without leaving the screen.
- Manager can inspect why an action was blocked or requires approval.
- UI still feels Bukeer when fixtures are replaced by read-only data.
