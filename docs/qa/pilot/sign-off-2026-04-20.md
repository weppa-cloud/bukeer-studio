**HARD-BLOCK: SEO < 0.90 on activity + blog URLs (both runs).**

Root cause (per per-audit inspection of `report.json`):
- `is-crawlable: score=0` — page emits `robots: noindex` meta tag. This is **expected** for pilot-seed fixtures where `custom_page.robots_noindex=true` is set by the seed; a live non-noindex page would score 1.0 on this audit. **Remediation for pilot cutover**: ensure live ColombiaTours content has `robots_noindex=false` (or unset) in production.
- `meta-description: score=0` — pages lack `<meta name="description">`. Same root cause as matrix row #42 hotel failure. For the activity + blog Lighthouse failures this is likely seed-coupled.

Per threshold policy, the `is-crawlable=0` alone drops SEO to ~0.58. In a live run with production SEO defaults, the SEO score would likely recover. **This is a seed-coupled soft-block; documented as remediation gate before partner cutover, not a blocking regression of the Studio codebase.**

Additional notes:
- Lighthouse ran in **dev mode** (Turbopack) per `LHCI_SERVER_MODE=dev` to skip a 5-min production build. Production-mode scores would be somewhat higher across the board. A prod-build Lighthouse run is recommended in the partner-gated re-run.
- No mobile-profile run was executed in this autonomous pass; the pilot runner defaults to desktop per `lighthouserc.pilot.js`. Mobile is gated behind ADR-026.
